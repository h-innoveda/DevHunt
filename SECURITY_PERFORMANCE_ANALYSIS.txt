2.1 BLOCKING DATABASE OPERATIONS (MAJOR)
─────────────────────────────────────────
ISSUE:
  Current: SQLite with synchronous reads/writes
  Problem: Every API call blocks waiting for database
  Impact: 100 concurrent users = database lock contention
  
  Files Affected:
    - backend/app.py: Every endpoint queries SQLite
    - backend/core/rag_pipeline.py: Vector search is slow
    - backend/core/chat_engine.py: Message logging blocks response

SOLUTIONS:

  A) Database Connection Pooling:
     Install: pip install SQLAlchemy
     
     from sqlalchemy import create_engine
     from sqlalchemy.pool import QueuePool
     
     engine = create_engine(
         'sqlite:///devhunt.db',
         poolclass=QueuePool,
         pool_size=10,
         max_overflow=20,
         pool_recycle=3600
     )

  B) Async Database Operations:
     Install: pip install asyncpg databases
     
     Use async with context:
       async with database.transaction():
           await database.execute(query)
     
     This prevents blocking I/O

  C) PostgreSQL Instead of SQLite:
     SQLite locks entire database on write
     PostgreSQL allows concurrent writes
     
     Benefits: 10x faster, supports 1000+ concurrent users
     Installation: docker run postgres:15

PRIORITY: CRITICAL - biggest performance bottleneck


2.2 BLOCKING API CALLS (MAJOR)
───────────────────────────────
ISSUE:
  Current: Synchronous requests to:
    - Google Gemini API (embedding, chat)
    - External IP lookup APIs (ipapi.co, ip-api.com)
    - URL fetching for RAG (requests.get())
    - YouTube downloads (yt-dlp)
  
  Each request waits 1-10 seconds, blocking other users

SOLUTION - Use Async/Await:

  Install: pip install aiohttp asyncio
  
  # Async Gemini API call
  async def get_embedding_async(text):
      loop = asyncio.get_event_loop()
      return await loop.run_in_executor(
          None, 
          get_embedding, 
          text
      )
  
  # Async external API
  async def get_public_ip_async():
      async with aiohttp.ClientSession() as session:
          async with session.get("https://ipapi.co/json/") as resp:
              return await resp.json()

QUICK FIX (without rewriting):
  Use ThreadPoolExecutor for CPU-bound tasks:
  
  from concurrent.futures import ThreadPoolExecutor
  executor = ThreadPoolExecutor(max_workers=10)
  
  # In route handler
  future = executor.submit(blocking_function, args)
  result = future.result(timeout=30)

2.4 INEFFICIENT FILE OPERATIONS (MEDIUM)
──────────────────────────────────────────
ISSUE:
  Current: os.listdir() called repeatedly
  
  Example - backend/app.py, music_list():
    for f in sorted(os.listdir(MUSIC_DIR), key=lambda x: os.path.getmtime(...)):
  
  Problem: stat() called for EVERY file in directory, then sorted
  If 10,000 music files: 10,000 disk I/O operations!

SOLUTION - Cache Directory Listings:

  from functools import lru_cache
  from datetime import datetime, timedelta
  
  class DirectoryCache:
      def __init__(self, ttl_seconds=60):
          self.cache = {}
          self.ttl = ttl_seconds
      
      def list_files(self, directory):
          now = datetime.now()
          if directory in self.cache:
              cached_data, timestamp = self.cache[directory]
              if (now - timestamp).seconds < self.ttl:
                  return cached_data
          
          files = sorted(
              os.listdir(directory),
              key=lambda x: os.path.getmtime(os.path.join(directory, x)),
              reverse=True
          )
          self.cache[directory] = (files, now)
          return files
  
  dir_cache = DirectoryCache(ttl_seconds=30)
  files = dir_cache.list_files(MUSIC_DIR)

IMPACT: 100-1000x faster directory listing

  // Cache API responses in localStorage
  const CACHE_TTL = 5 * 60 * 1000;  // 5 minutes
  
  async function fetchWithCache(url) {
      const cached = localStorage.getItem(url);
      if (cached) {
          const { data, timestamp } = JSON.parse(cached);
          if (Date.now() - timestamp < CACHE_TTL) {
              return data;
          }
      }
      
      const response = await fetch(url);
      const data = await response.json();
      localStorage.setItem(url, JSON.stringify({
          data,
          timestamp: Date.now()
      }));
      return data;
  }

Benefits:
  - 90% faster repeat requests
  - Reduced server load
  - Works offline for cached data

6.1 PAGINATION FOR LARGE DATASETS
──────────────────────────────────

Current Issue:
  /api/knowledge returns ALL sources every time

Solution:

  @app.route('/api/knowledge', methods=['GET'])
  def list_knowledge_sources():
      page = request.args.get('page', 1, type=int)
      per_page = request.args.get('per_page', 20, type=int)
      
      offset = (page - 1) * per_page
      
      cursor.execute("""
          SELECT id, name, type, path, status, chunk_count, created_at 
          FROM knowledge_sources 
          ORDER BY created_at DESC
          LIMIT ? OFFSET ?
      """, (per_page, offset))
      
      sources = [dict(r) for r in cursor.fetchall()]
      
      cursor.execute("SELECT COUNT(*) as total FROM knowledge_sources")
      total = cursor.fetchone()['total']
      
      return jsonify({
          'success': True,
          'sources': sources,
          'pagination': {
              'page': page,
              'per_page': per_page,
              'total': total,
              'pages': (total + per_page - 1) // per_page
          }
      })

Benefits:
  - Load only 20 items instead of 10,000
  - Faster API response
  - Better frontend UX (lazy loading)

7.1 MIGRATE FROM SQLITE TO POSTGRESQL
──────────────────────────────────────

Why PostgreSQL is better:
  - Concurrent writes (SQLite locks entire DB)
  - Native indexing options
  - Query optimization (EXPLAIN ANALYZE)
  - Connection pooling
  - JSONB for semi-structured data
  - Full-text search

Steps:

  1. Install PostgreSQL:
     docker run -d \
       -e POSTGRES_PASSWORD=devhunt_password \
       -p 5432:5432 \
       postgres:15

  2. Update requirements.txt:
     pip install psycopg2-binary SQLAlchemy

  3. Create connection (backend/config.py):
     import os
     from sqlalchemy import create_engine
     
     DATABASE_URL = os.getenv(
         'DATABASE_URL',
         'postgresql://postgres:devhunt_password@localhost:5432/devhunt'
     )
     
     engine = create_engine(
         DATABASE_URL,
         pool_size=20,
         max_overflow=40,
         pool_recycle=3600
     )

  4. Migrate existing data:
     # Dump SQLite
     sqlite3 backend/data/devhunt.db .dump > backup.sql
     
     # Load into PostgreSQL
     psql -h localhost -U postgres -d devhunt < backup.sql

  5. Benefits observed:
     - 10x faster concurrent writes
     - ACID transactions
     - Better crash recovery

7.3 QUERY OPTIMIZATION
───────────────────────

Use EXPLAIN ANALYZE to find slow queries:

  EXPLAIN ANALYZE SELECT * FROM messages WHERE session_id = ?;

Common Optimizations:

  # Bad: Full table scan
  SELECT * FROM messages WHERE LOWER(role) = 'user';
  
  # Good: Use index
  SELECT * FROM messages WHERE role = 'user';
  
  # Bad: Function on indexed column
  SELECT * FROM messages WHERE YEAR(timestamp) = 2024;
  
  # Good: Range query on index
  SELECT * FROM messages WHERE timestamp >= '2024-01-01' AND timestamp < '2025-01-01';
